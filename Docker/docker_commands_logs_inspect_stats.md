# Docker Monitoring and Inspection Commands

Understanding how to monitor and inspect Docker containers is crucial for debugging, auditing, and maintaining containerized applications. This guide covers three essential Docker commands: `docker logs`, `docker inspect`, and `docker stats`.

---

## 1. `docker logs`

The `docker logs` command is used to view the console output (both `stdout` and `stderr`) generated by a container. It is the first place you should look when an application in a container crashes or behaves unexpectedly.

### Basic Usage
```bash
docker logs <container_id_or_name>
```

### Useful Flags
*   `-f` or `--follow`: Follow log output in real-time (behaves like Linux `tail -f`).
*   `--tail <number>`: Print only the last `<number>` of lines from the logs.
*   `-t` or `--timestamps`: Show exact timestamps for each log entry.
*   `--since`: Show logs since a specific timeframe (e.g., `--since 30m` for the last 30 minutes).

### Examples
```bash
# Watch the logs of 'web_server' live, starting with the last 20 lines
docker logs -f --tail 20 web_server

# See what happened in a container in the last 15 minutes with timestamps
docker logs --since 15m -t web_server
```

## 2. `docker inspect`

The `docker inspect` command returns detailed, low-level configuration and state information about Docker objects (containers, images, volumes, networks) in **JSON format**. 

### Basic Usage
```bash
docker inspect <container_id_or_name>
```
*(This will dump a massive JSON array structure with everything Docker knows about the container.)*

### Extracting Specific Information (`--format`)
Because the output is huge, you usually want to filter it. Docker allows you to use the `--format` flag with Go templates to extract exactly what you need.

### Examples
```bash
# Get the internal IP address of a container
docker inspect --format='{{range .NetworkSettings.Networks}}{{.IPAddress}}{{end}}' db_postgres

# Check if a container is currently running (returns true or false)
docker inspect --format='{{.State.Running}}' my_app

# Find the path of a container's log file on the host OS
docker inspect --format='{{.LogPath}}' my_container
```

## 3. `docker stats`

The `docker stats` command provides a live stream of container runtime resource usage statistics. It is the Docker equivalent of the Linux `top` or `htop` command.

### Basic Usage
```bash
docker stats
```
*(Running this without specifying a container will show a live-updating table for ALL running containers).*

### Metrics Provided:
*   **CPU %:** Percentage of the host's CPU the container is consuming.
*   **MEM USAGE / LIMIT:** Amount of RAM used versus the maximum allowed limit.
*   **MEM %:** Percentage of memory utilized.
*   **NET I/O:** Amount of data sent and received over the network.
*   **BLOCK I/O:** Amount of data read from and written to the disk/storage.
*   **PIDS:** Total number of processes and threads running inside the container.

### Useful Flags
*   `--no-stream`: Get a single point-in-time snapshot of the stats rather than a continuous live stream (useful for scripting).
*   `-a` or `--all`: Include stopped containers in the output.

### Examples
```bash
# See live stats for two specific containers
docker stats frontend backend

# Get a one-time snapshot of resource usage for all running containers
docker stats --no-stream
```
