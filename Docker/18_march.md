# Basic Docker Commands

## Running and Managing Containers

### `docker pull nginx`
Pulls the `nginx` image from the registry to your local machine.

### `docker run -d -p 80:80 --name mynginx nginx`
Runs the `nginx` container in the background and exposes it to the browser.

**Purpose of flags:**
*   `-d` → runs the container in the background (detached mode), keeping your terminal free.
*   `-p` → maps ports (host:container), used for networking so you can access the container.
*   `--name` → assigns a custom, memorable name to the container instead of a random string.

---

## Basic Lifecycle Commands

### `docker ps`
Shows a list of currently running containers.

### `docker stop mynginx`
Stops the running container gracefully.

### `docker start <container_id>`
Starts a previously stopped container.

### `docker rm -f mynginx`
Removes a container forcefully (even if it is currently running). Use cautiously.

### `docker rmi nginx`
Removes a Docker image from your local system.

---

## Other Important Commands

### `docker exec -it <container_id> bash`
Opens an interactive terminal (`bash`) inside a running container, allowing you to run commands inside it.

### `docker logs <container_id>`
Shows the container's output/logs (useful for debugging).

### `docker inspect <container_id>`
Shows detailed, low-level configuration of the container in JSON format.

---

## Volume Commands
Volumes are used to persist data generated by and used by Docker containers.

### `docker volume create myvolume`
Creates a new persistent storage volume.

### `docker volume ls`
Lists all Docker volumes on your system.

### `docker volume inspect myvolume`
Shows detailed information about a specific volume.

### `docker volume rm myvolume`
Removes a volume (note: the volume cannot be currently in use by a container).
